# Рядовой Авангард — Telegram Mini App

Каркас («пустырь») мини-приложения Telegram. Концепция — позже.
Сейчас это пустой, но рабочий скелет, который открывается внутри Telegram.

## Стек
- React 19 + TypeScript + Vite
- Telegram WebApp SDK (`telegram-web-app.js`, подключён в `index.html`)
- Тема Telegram прокидывается в CSS-переменные `--tg-*` (см. `src/telegram.ts`)

## Запуск локально
```bash
pnpm install
pnpm dev
```
Откроется на http://localhost:5173 . В обычном браузере покажет заглушку
«Открой внутри Telegram» — это нормально (там нет `window.Telegram`).

## Открыть внутри Telegram (для разработки)
Telegram требует **https**. Локальный сервер пробрасываем туннелем:
```bash
cloudflared tunnel --url http://localhost:5173
```
Команда выдаст адрес вида `https://xxxx.trycloudflare.com`.
Этот адрес вставляем в BotFather (см. ниже). URL живёт, пока запущен туннель,
и меняется при перезапуске — это только для разработки.

## Привязка к боту (BotFather)
1. Шаг `/newapp` → «send me the Web App URL».
2. Вставить https-адрес туннеля (или адрес постоянного хостинга).
3. Готово — Mini App открывается из карточки бота / прямой ссылки.
   Позже кнопку меню можно настроить: `/mybots` → Bot Settings → Menu Button.

## Безопасность
- **Bot token — секрет.** Хранить только на бэкенде / в `.env` (он в `.gitignore`).
  Никогда не клади его в фронтенд и в переменные `VITE_*` — они попадают в бандл.
- Если токен где-то засветился — отозвать через BotFather `/revoke`.
- `initDataUnsafe` на фронте использовать только для UI; доверять данным можно
  лишь после проверки `initData` (HMAC по токену) на сервере.

## Постоянный хостинг (когда будет готова концепция)
Это статика (`pnpm build` → `dist/`). Варианты бесплатного https:
- **Cloudflare Pages / GitHub Pages / Vercel** — бесплатно, удобно для статики.
- **Amvera** — если понадобится ещё и бэкенд (бот, проверка initData);
  деплой через `git push`, https выдаётся автоматически.
